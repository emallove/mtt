Timestamp (GMT):

The time-zone independent timestamp for when the test started.

Timestamp aggregation:

To see timestamps for each row use this menu.

Phase:

MTT submits results to the database on three of its phases.

  1. MPI Install

    The configuring and building of the MPI library.

  2. Test Build

    The compiling of the individual tests with the Open MPI wrapper compilers.

  3. Test Run

    The running of the individual tests with the ORTE.

Result:

Tests have only two possible outcomes: pass or fail.

Selections

Aggregation

To generate more generalized reports (meaning fewer rows and columns), use the 'Aggregate' checkboxes. Checking a selection as 'Aggregate' will remove it from the output table, counting the number of rows collapsed by the aggregation.

Filters

The textfield filters are categorized into phase independent, test run filters, and MPI install/test build filters.

Phase Independent

The following fields are present in all MTT phases.

Stderr:

Everything sent to stderr by the test case.

Stdout:

Everything sent to stdout by the test case.

Env:

Environment variables set in the ini file for the given test case.

Config args:

The configure arguments supplied in the in file for the MPI Install.

Test msg:

The result message output by the test.

Settings

Count

By test case

A test case is the testing of a single command (e.g., configure, mpirun, mpicc, etc.).

By test run

A test run is the testing of series of test cases in a single MTT run. An MTT test run "pass" is a series of "MTT test cases" belonging to a single MTT test run that do not contain a single test case failure.

Debug

Saving queries

Each query can be saved by bookmarking the link labeled "Link to this query" at the foot of each report.

Phase combining and implicit phase selection

The Reporter will combine phases if either 'All' has been selected from the Phase menu, or if the filters will permit this to happen. For example, if a phase-specific filter is being used (e.g., "Test name" does not exist in the MPI Install/Test Build phase, it only exists in the Test Run phase), then the report will only display results for that phase.
